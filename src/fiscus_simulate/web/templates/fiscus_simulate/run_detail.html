Model limitations (V1). Flat-rate taxes; no mortality/morbidity; fixed spending mix; proportional withdrawals; no rebalancing; stylized return generator. A run is a set of possible futures, not a single probability of success.
{% endblock %} {% block glossary %}

Success criteria (all coincide in V1, diverge in V2): Overall success = every criterion holds on a path. Plan fully funded = planned spending is met every quarter (no shortfall). Portfolio never negative = net worth ≥ 0 every quarter. Terminal above threshold = ending net worth exceeds the configured floor. Each rate is the fraction of scenarios satisfying it.

Terminal / minimum net worth are nominal (of-the-day) sums across all accounts at the final quarter / at the worst quarter. Divide by the inflation deflator for real (today's-money) terms.

Taxes paid = summed nominal tax over all 160 quarters: interest & dividends on the taxable account, the taxable part of pension income, tax-deferred withdrawals (taxed on withdrawal), and realized capital gains (via cost basis). Assets sold = gross proceeds raised by selling to fund spending.

Percentiles vs. terminal-ranked. In the Percentiles view each column is sorted on its own, so the p50 of taxes and the p50 of terminal wealth are different households. In By terminal net worth the scenarios are ranked by terminal net worth and each row is that one household's actual figures — so the row reads across coherently. The mean row is identical in both (averages don't depend on ordering).
